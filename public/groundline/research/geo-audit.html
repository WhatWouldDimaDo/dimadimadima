AI-visibility (GEO) audit — methodology

← back to research

What this is: the repeatable method behind the pitch line "AI recommends your biggest competitor 23× more often than you." One audit = one trade × one territory × one client company, run across 6 AI engines, scored, and delivered as a one-page audit card that slots into the "spotlight on them" section of the one-sheet.

Why it's an open wedge: none of the four profiled competitors (CurbLead, Porchline, PostScout, RenderMail) sell any diagnostic/visibility product. This is the only artifact in the space that makes the problem visible before selling the fix.

Established mechanics (do not re-derive — from the Jul 16 SEO/AI-search research):

  • Google AI Overviews cite only what already ranks top-10/20 organically.
  • Perplexity indexes fastest, citations are transparent.
  • ChatGPT leans on aggregate third-party reputation (listicles, review platforms, forum mentions).
  • FAQPage schema = highest-leverage structured data for GEO.
  • Third-party citations beat owned-site content for AI answers.

These mechanics are why the audit is predictable from organic footprint — and why the remediation roadmap writes itself from the citation-source breakdown (section 3).

1 · Query battery (parameterized, ~18 queries per trade/territory)

Variables: {trade} (landscaping / pool service / driveway paving / roofing), {job} (trade-specific project noun), {city}, {hood1}, {hood2} (two target neighborhoods/suburbs), {company} (the client), {competitor} (top local rival, from the SERP winners).

Discovery intent (the money queries — weight 2× in scoring)

#Template
D1best {trade} company in {city}
D2best {trade} in {hood1}
D3best {trade} in {hood2}
D4{trade} near me — I'm in {hood1} (location stated in-prompt, see section 2)
D5who should I hire to {job} in {city}?
D6recommend a high-end {trade} contractor in {hood1}, budget is not the constraint
D7I need a {trade} company that does large projects ($25K+) in {city}
D8top rated {trade} companies {city} 2026

Job-specific intent

#Template
J1how much does {job} cost in {city} and who does good work?
J2best company for {job} in {hood1}
J3{job} contractor recommendations {city}

Validation intent (homeowner checking a name they already have)

#Template
V1is {company} legit?
V2{company} {city} reviews
V3{company} vs {competitor} — which is better?
V4has anyone used {company} in {hood1}?

Comparative / list intent

#Template
C1give me 5 {trade} companies to get quotes from in {city}
C2who are the top 3 {trade} companies serving {hood1} and {hood2}?
C3which {trade} companies in {city} have the best reviews?

Trade fills for {job}: landscaping → "a full landscape redesign"; pools → "a pool renovation" / "weekly pool maintenance" (run both if the client does both); driveways → "replace my concrete driveway"; roofing → "replace my roof".

Rule: keep the wording of each template fixed across companies and across time — the battery is only comparable if the prompts are identical. Version the battery (this is v1); any wording change bumps the version and breaks comparability with prior runs.

Honest automation summary: Perplexity (API) and Google AIO (via a SERP API vendor) are the two engines worth automating first. ChatGPT/Claude/Gemini/Grok APIs are fine for trend monitoring at scale later, but for the sales artifact the consumer app is the ground truth — it's what the customer's homeowner actually types into. For the demo artifact: run the consumer apps, screenshot everything. The screenshot is the product — "here is what a Dunwoody homeowner sees when they ask ChatGPT" beats any chart.

Run protocol (per engine, per query)

  1. Fresh session every query. New temporary/incognito chat; memory and personalization off; no logged-in history. One query per chat — never ask follow-ups in the same thread (context contaminates).
  2. Repetitions: n = 3 per query per engine. LLM answers vary run-to-run; n=3 is the floor that separates "always mentioned" from "got lucky once." (n=5 on the 8 discovery queries if time allows — they carry 2× scoring weight.) Aggregate = mention counted if it appears in ≥1 of the runs; mention rate computed over all runs.
  3. Record per run (one spreadsheet row per engine × query × run):
    • mentioned (client named at all): Y/N
    • rank (position in the answer's list, 1-based; blank if unranked prose)
    • sentiment: positive / neutral / negative / hedged ("mixed reviews")
    • competitors_named (all of them, in order)
    • cited_sources (URLs/domains the engine cites — Perplexity and AIO give these directly; ChatGPT/Gemini sometimes; record "none shown" otherwise)
    • screenshot_ref (filename; every run gets a screenshot, demo pulls the best 3-4)
    • no_answer flag (engine refused, or AIO didn't trigger)
  4. Timing: run all engines within the same 48-hour window; date-stamp the audit. Answers drift — an audit is a snapshot, which is also the re-sell: quarterly re-runs are the retainer hook.
  5. Hygiene: clean browser profile per engine, VPN off (or fixed to the client's metro), no autocomplete acceptance — type the query verbatim from the battery sheet.

Total run volume per audit: ~18 queries × 6 engines × 3 runs = ~324 runs (~360 with n=5 on discovery). This is the cost driver — see section 6.

3 · Scoring model

All scores computed from the run spreadsheet. Definitions are exact so the headline number is defensible in the room.

  • Mention rate (per company, per engine): runs where company is named ÷ total runs on that engine. Overall mention rate = weighted mean across engines, discovery queries counted 2×.
  • Share of voice (SoV): for each run, the named companies split 1 point by inverse rank (rank 1 = 1.0, rank 2 = 0.5, rank 3 = 0.33, unranked prose mention = 0.25). SoV(company) = company's points ÷ all points awarded across all runs. This rewards being first-named, which is what converts.
  • The "23×" number — Competitor Multiple: mentions(top competitor) ÷ mentions(client), where mentions = total runs naming the company across all engines and queries, and top competitor = the single most-mentioned company in the battery. Rules for defensibility: (a) if client mentions = 0, report the multiple as "{N}× vs. never — {competitor} was named {N} times across {M} AI answers; {client} was named zero times" — never divide by zero, the honest phrasing is stronger anyway; (b) always name the denominator ("across 324 AI answers on 18 homeowner questions, July 2026"); (c) the multiple is engine-blended — keep the per-engine table in the appendix so it survives scrutiny.
  • Citation-source breakdown = the remediation roadmap. Tally cited_sources domains across all runs → ranked table of which third-party sites actually feed the AI answers in this trade/territory (expected heavy hitters per the established mechanics: Yelp, Houzz, Angi/HomeAdvisor, Thumbtack listicle pages, Reddit/Nextdoor threads, local-news "best of" lists, Birdeye-syndicated review pages, the winners' own service-area pages). For each top source domain, mark: is the client present there at all? This table converts directly into the ranked fix list — you don't guess where to build presence, the engines tell you.
  • Validation-query health (V1–V4): scored separately as pass/warn/fail — does the AI confirm the client exists and is reputable when asked by name? A referral-based business fails silently here: the homeowner got the name from a neighbor, asked ChatGPT "is X legit," and got "I couldn't find much information." That's a lost $50K job with no trace.

4 · Deliverable artifact spec — the audit card

One page (PDF / printed card), per company. Slots into the one-sheet's "spotlight on them" section. Layout top to bottom:

  1. Verdict line (one sentence, the cold shower): "When Dunwoody homeowners ask AI who should redo their yard, {competitor} gets named {N}× — you get named {zero/once}." Date-stamped, with the denominator in small print.
  2. Share-of-voice bar — single horizontal stacked bar: top 5 companies by SoV + "everyone else," client's slice in red (or an arrow marking zero). One chart, no dashboard.
  3. Evidence strip — 2-3 cropped consumer-app screenshots (ChatGPT + Perplexity + AI Overview) showing the competitor being recommended for the client's exact territory. This is the piece that makes it real; the numbers alone don't.
  4. "Where the AI gets its answers" — the top 5-7 cited source domains from section 3, each marked present/absent for the client. This reframes the problem from "AI is a black box" to "you're absent from 6 specific websites."
  5. 3 remediation moves, ranked by leverage — pulled from the citation table + the established mechanics, in fixed priority logic: (1) whatever gets them into the top-cited review source (usually GBP claim + review velocity — feeds AIO via organic and ChatGPT via aggregate reputation); (2) presence on the top-cited listicle/directory source (Houzz/Angi/local "best-of"); (3) owned-site structural fix (service-area page + FAQPage schema). Each move: effort tag (hours/weeks) + which engines it moves.
  6. Footer: method note ("{X} AI answers across 6 engines, {date range}, identical homeowner questions") + re-audit offer (quarterly).

Production rule: the card is generated from the run spreadsheet + screenshot folder — template it once (HTML → PDF), so audit #2 is assembly, not design.

5 · Demo groundwork — organic footprint check (verified via web search, 2026-07-23)

Per the established mechanic — AI cites what ranks and what third parties say — organic footprint predicts the engine-run outcome. Both entries below are PREDICTED from organic footprint; actual engine runs (section 2) still need manual execution and are the demo build step.

Creekside Premier Landscaping (Dunwoody/Buckhead GA)

Footprint (from the 2026-07-16 Track A audit, spot-checked today — unchanged): no discoverable GBP/review count; absent from Yelp, Houzz, Angi, Thumbtack, Nextdoor; not in top-10 organic for "landscaping Dunwoody GA" or "landscape design Buckhead"; Wix site with no keywords/schema; FB + Instagram exist but surface no reviews. Today's "Creekside Premier Landscaping" reviews search returns only same-named companies in OH/CA/CO — zero review presence for the Atlanta business; only its own site, Instagram, and Facebook page appear.

PREDICTED audit outcome (label: predicted-from-organic-footprint, pending engine runs):

  • Discovery queries (D1–D8): ~0% mention rate across all six engines. Nothing to cite — no reviews, no listicles, no directory pages, no ranking service-area pages. Expected winners: Bloom'n Gardens, White Oak, RelyNow, Buckhead Landscape & Design (the SERP/map-pack incumbents from Track A).
  • Validation queries (V1–V4): fail — engines likely to say they can't find substantive information, and possibly confuse it with the unrelated "Premier Landscaping" (16 Yelp reviews) or the Sacramento/Ohio Creeksides. Brand-confusion risk is itself a card-worthy finding.
  • Headline shape: "{top competitor} named {N}×, Creekside named zero — {N}× vs. never."
  • This makes Creekside the perfect internal demo — the before/after will be maximal — and the card doubles as the GEO-retainer justification for the client conversation.

North Atlanta Pool Services (Atlanta GA — Wieuca Rd, Buckhead-adjacent)

Footprint (verified today): real 38-year business (founded Dec 1987, Jim Morgan) with a genuine third-party citation trail — Yelp listing (rating/count not retrievable, Yelp blocked fetch; appears low-volume), Angi 4.5-star, BBB A+, HomeAdvisor, Nextdoor page, Yahoo Local. But: website is napsi.com — non-keyword domain, HTTP, legacy build; and the company is absent from every "best pool service Atlanta" surface that ranks — the Swimply top-10, Yelp's "best" SERP, Houzz top-15, and the award-badged players (Pinnacle "Best Pool Cleaning 2025," Anthony & Sylvan "Forbes Best of 2024," Pool Troopers, Sears Pool) that dominate the money query.

PREDICTED audit outcome (label: predicted-from-organic-footprint, pending engine runs):

  • Discovery queries: low but nonzero — occasional mention on citation-transparent engines (Perplexity may surface via Angi/BBB/HomeAdvisor), routinely beaten on "best pool service Atlanta" by the listicle/award incumbents. Expected SoV: low single digits vs. Pinnacle / Anthony & Sylvan / Pool Troopers.
  • Validation queries: pass/warn — 38 years + BBB A+ + Angi 4.5 should let engines confirm legitimacy, though thin review volume may produce hedged answers.
  • Headline shape: a true multiple, not "vs. never" — e.g. "{award-badged competitor} named {N}× more often." This is the more typical prospect profile: good real-world reputation, invisible to AI on discovery intent — i.e., the exact customer the wedge is built for. Remediation story is clean: get onto the cited listicle/award surfaces + fix the owned site; the reputation substance already exists.

Demo pairing logic: Creekside shows the floor (invisible), North Atlanta Pool shows the norm (established-but-losing). Run both engine batteries in one sitting → two contrasting cards for the pitch deck.

6 · Effort / cost estimate

One full manual audit (v1, today):

StepHours
Battery instantiation (fill variables, identify top competitor from a pre-check)0.5
Engine runs: ~324 runs, manual, fresh sessions, screenshots (~1 min/run incl. logging)5.5–6.5
Scoring spreadsheet + citation tally1.0
Card production (after template exists; first-ever card add +2 for template build)1.0
Total per audit~8–9 hrs (first one ~11 with template build)

At the pilot's $1,500/mo anchor, the audit prices naturally at $500–$1,500 standalone (or free-as-wedge for whale prospects) — a few hours of margin either way, and it front-loads the retainer conversation.

Automation path (later, in order of ROI):

  1. Perplexity Sonar API runner — full battery + citations in minutes, ~$1-2 in API cost. Cuts ~1.5 hrs and gives the citation table for free. Build first.
  2. SERP API for AI Overviews (SerpAPI or similar, ~$50/mo tier) — automates the Google layer compliantly.
  3. Scoring sheet → card pipeline (script the tally + HTML-to-PDF card). Cuts ~1.5 hrs.
  4. ChatGPT/Claude/Gemini API runs as a monitoring layer for quarterly re-audits (trend lines are fine from APIs; the sales artifact still uses consumer-app screenshots).
  5. End state: manual work drops to ~2 hrs/audit (consumer-app screenshot passes + review), which is what makes the quarterly re-audit retainer economics work at 5+ clients.

Not worth automating: the consumer-app runs themselves via browser automation — ToS-fragile, and the screenshots need human judgment on what's demo-worthy anyway.
